# 요구사항 정의서 – MEMENTO

## 1. 목적
사용자의 사망 또는 장기 미접속을 자동 감지하여  
디지털 자산 및 사후 메시지를 안전하게 관리하는 시스템.

---

## 2. 주요 기능 요구사항

### 2.1 사용자 관리
- 회원가입 / 로그인 / JWT 인증
- 사용자 정보 조회/수정
- 역할(ADMIN / USER) 구분

### 2.2 디지털 자산 관리
- 자산 추가/삭제/조회
- 서비스명 / 카테고리 / 로그인 ID / 메모 관리
- 사후 처리 지시(DELETE / TRANSFER / KEEP 등)

### 2.3 신뢰 연락처
- 신뢰 연락처 등록 (최소 1명)
- 사망 의심 신고 기능
- 연락처 검증(3명 중 과반 동의 → CONFIRMED)

### 2.4 타임캡슐(사후 메시지)
- ON_DEATH : 사망 확정 후 자동 공개
- ON_DATE : 특정 날짜 자동 공개
- IMMEDIATE : 즉시 공개

### 2.5 자동 로직 (Scheduler)
- 72시간 후 자동 사망 확정
- ON_DATE 메시지 지정 날짜 자동 공개

---

## 3. 비기능 요구사항
### 성능
- REST API 응답 200ms 이하
- 정기 스케줄러 안정성 보장

### 보안
- JWT 기반 인증
- 비밀번호 bcrypt 해시화
- .env로 환경변수 분리

### 유지보수성
- controllers / services / routes 구조 분리
- GitHub 협업 규칙 준수
